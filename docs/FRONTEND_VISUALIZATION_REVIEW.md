# 前端可视化审查报告

## 审查范围
本次审查聚焦项目中面向用户的可视化/推理前端：

- `app_platform.py`（基于 Streamlit 的多模型推理界面）
- `step12_infer_and_visualize.py`（离线可视化与实验推理流程）

---

## 当前前端概况

`app_platform.py` 已具备以下基础能力：

- 本地模型仓多模型切换（`app_platform.py:40`）
- 多输入方式：上传文件 / 本地 `.mhd` 路径 / 数据集样本（`app_platform.py:351`）
- 三视图叠加预览（轴状/冠状/矢状）（`app_platform.py:264`）
- 推理结果持久化与掩码下载（`app_platform.py:288`，`app_platform.py:436`）

整体上已经可以作为可部署演示平台的基础版本。

---

## 主要问题（按严重级别排序）

### 1）高：不支持上传 5D 张量
- **定位：** `app_platform.py:177`
- **问题：** `_to_3d()` 仅支持 2D/3D/4D，常见医学张量如 `(N,1,D,H,W)` 或 `(1,D,H,W,1)` 会报错。
- **影响：** 上传项目自有整包数据（例如 `(600,1,64,64,64)`）可能直接失败。
- **建议：** 扩展 `_to_3d()` 的 5D 解析逻辑，显式选择样本与通道并做形状校验。

### 2）高：前端推理路径与实验最优流程不一致
- **定位：** `app_platform.py:255`，对照 `step12_infer_and_visualize.py:106`
- **问题：** 当前前端只做固定阈值二值化，而你的实验主流程还包含：
- `TopK` 自适应阈值选择
- 不确定性抑制（`entropy -> suppress_prob`）
- **影响：** 前端结果可能弱于论文/实验结果，且两者不一致。
- **建议：** 在 UI 增加开关并复用现有逻辑：
- `使用自适应阈值`
- `使用不确定性抑制`

### 3）中：结果目录存在同秒覆盖风险
- **定位：** `app_platform.py:296`
- **问题：** 输出目录仅使用秒级时间戳；同一秒重复点击推理可能覆盖文件（`exist_ok=True`）。
- **影响：** 结果被静默覆盖，难以追溯。
- **建议：** `run_dir` 增加毫秒后缀或 UUID。

### 4）中：2D 图片上传通过切片复制伪造 3D
- **定位：** `app_platform.py:187`
- **问题：** PNG/JPG 会通过重复单张切片扩成 3D 体数据。
- **影响：** 预测结果看似可用，但医学意义较弱。
- **建议：** 在界面明确标注“演示模式”，或默认关闭该输入方式。

### 5）中：预处理丢失临床几何信息
- **定位：** `app_platform.py:194`，`app_platform.py:231`
- **问题：** 所有输入统一 resize 到 `(64,64,64)`，未暴露基于 spacing 的重采样策略。
- **影响：** 各向异性 CT 可能产生形态畸变，影响稳定性。
- **建议：** 为 `.mhd` 增加 spacing-aware 重采样选项，并显示原始 spacing。

### 6）低：数据集样本模式缺少 GT 对照
- **定位：** `app_platform.py:383`，`app_platform.py:418`
- **问题：** 已加载 `masks`，但未展示 GT 叠加。
- **影响：** 无法快速做可视化真值核验。
- **建议：** 增加开关：`显示 GT 叠加（仅数据集模式）`。

## 优点

- 模型注册结构清晰（`ModelConfig` + `MODEL_ZOO`）（`app_platform.py:31`，`app_platform.py:40`）
- 输入解析、模型加载、推理、可视化、保存职责分离较好
- 交互基础完备：加载提示、摘要指标、导出下载

---

## 建议迭代计划

### Phase 1（快速修复，1-2 小时）
- 增加 5D 上传支持
- 解决结果目录覆盖风险
- 增加 2D 伪 3D 输入提示

### Phase 2（结果一致性，0.5-1 天）
- 将 `step12_infer_and_visualize.py` 的自适应阈值与不确定性抑制并入前端
- 增加相关参数开关与配置项

### Phase 3（验证体验，0.5 天）
- 数据集模式增加 GT 叠加
- 增加概率直方图与阈值敏感曲线面板

---

## 结论

当前前端已经可用，但**尚未完整反映你在实验流程中的最优推理策略**。  
最高优先级是让前端推理逻辑与 `step12_infer_and_visualize.py` 保持一致，并增强真实数据输入的鲁棒性。
